Finch 技术架构

本地优先的 AI 桌面助手 — 架构概览

Finch 采用 Electron 双进程架构，主进程负责 Agent 编排和系统操作，渲染进程提供用户界面。两者通过 IPC 通信，所有跨边界数据类型在共享层统一声明。

渲染进程 ⬇ IPC ⬆ 主进程 ⬇ ⬆ Agent 核心 (pi) ⬇ ⬆ Electron 运行时

进程分层

渲染进程

基于 React + TypeScript + Zustand 构建的桌面 UI。组件按功能域组织，通过 desktopApi 桥调用后端能力，不走 Node 直接 API。全局状态按领域拆分独立 Store。

主进程

服务的容器。所有业务逻辑（Agent 编排、文件 I/O、会话持久化、模型管理）以服务层形式组织在独立模块中。IPC 处理层保持薄层，仅作为路由转发。

Agent 核心

运行在 ~/.finch/pi/ 目录下，管理会话生命周期、工具调度、模型通信。以 JSONL 格式持久化会话记录，按工作目录切分存储。

关键子系统

Agent Runner（Agent 运行器）

接收用户输入，调用大模型生成响应，编排工具执行。支持流式输出和后台任务。每次运行独立上下文，终态不互相覆盖。

流式响应 工具编排 后台任务

Memory System（记忆系统）

分层记忆架构：短期记忆注入会话上下文、中期记忆提炼项目摘要、长期记忆归档日常日志。自动蒸馏机制定期压缩和淘汰低价值内容。

分层记忆 自动蒸馏 容量控制

Skill System（能力框架）

将 AI 的能力封装为可注册、可触发的技能模块。每个 Skill 包含触发规则、执行上下文、权限声明。当前内置文档处理、PDF 编辑、PPT 制作、前端设计、问卷调查等。

插件化 触发匹配 可扩展

Permission System（权限系统）

三级权限模式：Ask（逐次确认）、Accept Edits（自动编辑）、Developer（开发者模式）。文件操作和受信命令自动放行，高危操作二次确认，支持批量拒绝。

分级控制 安全沙箱 路径拦截

Model Layer（模型接入层）

统一的模型接入抽象层，兼容 OpenAI API 协议。支持 Anthropic Claude、OpenAI GPT、Ollama 本地模型，以及任何兼容 OpenAI 协议的第三方服务。模型配置持久化存储。

多后端 统一接口 本地/云端

Session Management（会话管理）

按工作目录管理会话，JSONL 格式持久化。支持多窗口并发查看同一会话，运行状态在窗口间实时广播。会话可恢复、可搜索。

多窗口 实时同步 持久化

数据与运行时

所有运行时数据存储在用户目录下的 ~/.finch/ 中，包括 Agent 核心数据、会话记录、模型配置和工作区状态。不依赖外部服务，本地优先运行。

跨平台支持 macOS（ARM64 / Intel）和 Windows（64位 / 32位），通过 Electron 的跨平台能力实现一次构建多端发布。
